⚠️ Setup Needed: Missing Backend URL

The current BACKEND_API_URL points to a Google Apps Script Library/Editor URL or spreadsheet link instead of an active Web App execution (/exec) endpoint.

This will cause uploads to Google Drive and deletions (Cloud Purge) to fail with network errors (Failed to Fetch) as libraries do not accept direct API POST/GET traffic.

HOW TO CONNECT YOUR CLOUD VAULT BACKEND PROPERLY:

  1. Open your Google Apps Script project editor.
  2. Copy the customized Apps Script template code shown below into your editor.
  3. Click on Deploy (top-right corner of the Apps Script page).
  4. Select New deployment.
  5. Click the gear icon next to "Select type" and select Web app.
  6. Fill in the deployment parameters:
    • Description: Secure Document Vault Backend
    • Execute as: Me (your Google account) (Crucial so it has permission to write to your Drive/Spreadsheet!)
    • Who has access: Anyone (Crucial so this React-based Vault can send upload API requests!)
  7. Click Deploy. Google will ask for permission authorization. Complete it.
  8. Copy the generated Web App URL (it must end with /exec).
  9. Paste that exact /exec URL as the value for BACKEND_API_URL around line 4 of app.js.
